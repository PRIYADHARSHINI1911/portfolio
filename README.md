## 🚀 Quick Start

### Option 1: Local Development
1. Clone/download the portfolio files
2. Open `index.html` in your browser
3. The website will work immediately without any build process

### Option 2: Deploy Online

#### GitHub Pages (Free & Recommended)
1. Create a new repository on GitHub
2. Upload all files (index.html, styles.css, script.js)
3. Go to repository Settings → Pages
4. Select main branch as source
5. Your portfolio is now live at `https://yourusername.github.io/portfolio`

#### Netlify (Drag & Drop)
1. Visit [netlify.com](https://netlify.com)
2. Drag and drop your portfolio folder
3. Your site is instantly deployed

#### Vercel
1. Visit [vercel.com](https://vercel.com)
2. Import your GitHub repository
3. Deploy with one click

#### Self-Hosted
1. Upload files to your web server via FTP
2. Ensure `index.html` is accessible
3. Configure domain to point to your server

## 🎯 Customization Guide

### Update Personal Information

**1. Hero Section Title**
```html
<!-- In index.html, find and modify: -->
<h1 class="hero-title">Your Title Here</h1>
```

**2. Social Links**
```html
<!-- Update href in hero-socials section -->
<a href="https://your-linkedin-url">LinkedIn</a>
<a href="https://your-github-url">GitHub</a>
```

**3. Contact Information**
```html
<!-- Update in contact section and footer -->
<a href="mailto:your-email@example.com">your-email@example.com</a>
<a href="tel:+your-phone">+Your Phone</a>
```

### Modify Colors

Edit CSS variables in `styles.css`:
```css
:root {
    --primary-color: #6366f1;        /* Main color */
    --secondary-color: #ec4899;      /* Secondary color */
    --accent-color: #f59e0b;         /* Accent color */
    --dark-bg: #0f172a;              /* Background color */
}
```

### Update Skills, Experience, or Projects

1. **Add Skills**: Update skill tags in the skills section
2. **Update Experience**: Modify timeline items
3. **Add Projects**: Duplicate project-card div and update content

## 🐛 Troubleshooting

**Q: External links not working?**
A: Ensure URLs include `https://` or `http://`

**Q: Fonts not loading?**
A: Check internet connection. Google Fonts require live internet.

**Q: Animations not smooth on mobile?**
A: This is normal on lower-end devices. Consider reducing animations in CSS.

**Q: Form not sending emails?**
A: The form opens the default email client. For true email backend, consider:
- Formspree (formspree.io)
- EmailJS (emailjs.com)
- Custom backend
